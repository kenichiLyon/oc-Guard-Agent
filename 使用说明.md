# OpenClaw-Agent 使用说明

## 1. 项目定位

本项目名为 `openclaw-agent`，其中：

- `openclaw` 是核心诊断引擎
- `openclaw-agent` 是对 `openclaw` 的工程化封装
- CLI 与 MCP Server 都复用同一套五层诊断能力

当前 MCP 主工具名为：

- `openclaw_agent_diagnose`

兼容旧工具名：

- `openclaw_diagnose`

## 2. 推荐环境管理方式

本项目推荐使用 `uv` 管理 Python 版本、虚拟环境与依赖。

首次进入项目根目录后执行：

```bash
uv python pin 3.13
uv sync --extra dev
```

说明：

- `uv python pin 3.13` 会让项目固定使用 Python 3.13
- `uv sync --extra dev` 会根据 `pyproject.toml` 创建 `.venv` 并安装开发依赖

激活虚拟环境：

```bash
.venv\Scripts\activate
```

如果不想手动激活，也可以直接使用：

```bash
uv run python -m pytest -q
```

## 3. CLI 使用方式

### 3.1 使用项目同名入口

```bash
uv run openclaw-agent --config examples/openclaw.config.json --incident-file examples/incidents/pod_crashloop.json --format pretty
```

### 3.2 使用兼容入口

```bash
uv run openclaw --config examples/openclaw.config.json --incident-file examples/incidents/pod_crashloop.json --format pretty
```

### 3.3 直接以模块运行

```bash
uv run python -m openclaw --config examples/openclaw.config.json --incident-file examples/incidents/pod_crashloop.json --format json
```

## 4. MCP 启动方式

### 4.1 推荐使用项目同名入口

```bash
uv run openclaw-agent-mcp
```

### 4.2 兼容旧入口

```bash
uv run openclaw-mcp
```

### 4.3 直接用模块启动

```bash
uv run python -m openclaw.mcp_main
```

## 5. MCP 客户端配置示例

如果客户端支持 stdio MCP，可参考：

```json
{
  "mcpServers": {
    "openclaw-agent": {
      "command": "uv",
      "args": ["run", "openclaw-agent-mcp"],
      "cwd": "G:/your-path/OpenClaw-Agent",
      "env": {
        "OPENCLAW_CONFIG": "G:/your-path/OpenClaw-Agent/examples/openclaw.config.json"
      }
    }
  }
}
```

## 6. 测试方式

```bash
uv run pytest -q
```
